Comment gérer vos chemins sur un site Drupal

Submitted by Albert on Mon, 11/05/2012 - 20:46

À penser:

- Quels chemins on veut pour différents contenus
- Que se passe-t-il si un visiteur retire une partie d'un chemin (par exemple, il tombe sur k.o/nouvelles/2010/08/02/x-y-z, et il efface tout après 08, y a-t-il une vue ou page à prévoir ou le visiteur tombe-t-il sur une "page non trouvée")
- Liens permanents (genre k.o/nouvelles/2010/08/02/x-y-z) vs. temporaires (k.o/yxz) -- dans ce cas les deux iraient à la même place, mais xyz a une durée de vie temporaire
- Liens déjà sur le web (faire une recherche google site:www.koumbit.org ou site:koumbit.org) -- disparaîtront-ils lors de la mise en ligne du nouveau site?

À installer:

- Le module "path" de core
- Le module http://drupal.org/project/pathauto pour générer des chemins personnalisés selon certaines règles
- Le module http://drupal.org/project/pathologic, un filtre pour manipuler des URL dans des ondes
- Le module http://drupal.org/project/globalredirect qui gère des versions canoniques des chemins

Scénario à régler (pas sûr comment)

- On crée une "nouvelle" de type example.com/xy
- Ce chemin est référencé sur google
- On change le titre de la page et la nouvelle devient example.com/xyz
- Quelqu'un cherche xy sur google eet ça tombe sur Page not found
